# NIXKEY Agent Notes

## Read First

1. `AGENTS.md`
2. `README.md`
3. `extras/BRANDING.md`
4. `index.html`

## Hard Rules

- Preserve the existing NIXKEY brand direction.
- Use the current assets as source of truth:
  - `nixkey_logo_primary.svg`
  - `nixkey_favicon.svg`
  - `nixmintcard.png`
  - `extras/BRANDING.md`
- Keep the storefront premium and restrained.
- Prefer removing clutter over adding more UI.
- Current storefront is a static single-page site in `index.html`.
- Commerce is Snipcart-based.
- Do not remove Snipcart integration unless intentionally replacing it.
- Snipcart public key in `index.html` is still a placeholder and must be replaced manually.
- Every USB is currently:
  - `$19.99`
  - `32GB`
- Keep visible product content and Snipcart product attributes in sync.

## Catalog Structure

- Default catalog view is grouped into:
  - Beginner-Friendly
  - Creator & Gaming
  - Privacy & Security
  - Power User & Rolling
- Filter buttons should continue to work.
- Product lists should remain alphabetized within each category unless there is a strong reason not to.

## Current Priority Aesthetic

- Dark premium base
- Teal/orange accent system
- Clean hierarchy
- Minimal hero
- Low visual noise

## Avoid

- Marketplace-style clutter
- Overexplaining the implementation in UI copy
- Random new visual motifs that conflict with the existing packaging and logo work
- Breaking product grouping, pricing, or capacity rules without explicit instruction
